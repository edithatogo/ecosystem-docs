# 🌿 Ecosystem Documentation

Welcome to the **edithatogo ecosystem** — a family of open-source libraries and tools designed for **health economics and outcomes research (HEOR)**, **data science**, and **cross-domain computation**.

Each project in this ecosystem is independently useful, but they're designed to work together seamlessly. This central hub helps you discover, learn, and navigate across all projects.

---

## Projects

<CardGrid>
  <Card title="Innovate" icon="star">

## Ecosystem Principles

The edithatogo ecosystem is built on a set of shared principles:

| Principle | Description |
|-----------|-------------|
| **Interoperable by Design** | Projects share common data formats (Arrow, Parquet), naming conventions, and packaging strategies so they compose naturally. |
| **Polyglot from Day One** | APIs are designed to map cleanly across languages — Python for research, Go for production, TypeScript for the web. |
| **Health Economics First** | Each tool solves a real HEOR workflow: diffusion forecasting, value of information, patient-level simulation, and data pipeline management. |
| **Open Science** | All projects are MIT-licensed. Documentation is built with Starlight and deployed to GitHub Pages. Contributions welcome. |
