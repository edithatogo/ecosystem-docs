**Mars** is a Go toolkit for structured time-series data analysis. Designed for **production-grade data pipelines**, Mars provides efficient, concurrent-safe implementations of moving window operations, resampling, statistical decompositions, and signal processing primitives — all with a clean, composable API.

Whether you're processing sensor data, financial time series, or healthcare monitoring streams, Mars gives you the building blocks to analyse temporal data at scale.

## Quick Start

```go
package main

import (
    "fmt"
    "github.com/edithatogo/mars"
)

func main() {
    // Create a time series
    data := []float64{10, 12, 15, 14, 18, 22, 21, 25, 28, 30}

    // Compute a 3-period moving average
    ma := mars.RollingMean(data, 3)
    fmt.Println("Moving average:", ma)

    // Perform additive seasonal decomposition
    result := mars.Decompose(data, mars.Additive, 4)
    fmt.Printf("Trend: %v\n", result.Trend)
    fmt.Printf("Seasonal: %v\n", result.Seasonal)
    fmt.Printf("Residual: %v\n", result.Residual)

    // Resample from high to low frequency
    resampled := mars.Resample(data, mars.Downsample{
        Factor: 2,
        Method: mars.Mean,
    })
    fmt.Println("Resampled:", resampled)
}
```

## API Overview

| Function | Description |
|----------|-------------|
| `RollingMean(data, window)` | Moving average with configurable window size |
| `RollingStd(data, window)` | Rolling standard deviation |
| `RollingApply(data, window, fn)` | Custom rolling aggregation |
| `Resample(data, config)` | Frequency conversion with interpolation |
| `Decompose(data, model, period)` | Seasonal-trend decomposition (STL-compatible) |
| `ACF(data, maxLags)` | Autocorrelation function |
| `PACF(data, maxLags)` | Partial autocorrelation function |
| `ChangePoints(data, threshold)` | Change-point detection via PELT algorithm |
| `WindowedStream(data, window, step)` | Streaming window iterator |
