**Starlight Polyglot** is a plugin for [Astro's Starlight](https://starlight.astro.build/) that generates polyglot API documentation from multiple programming languages into a single, unified documentation site. One config, many languages.

It powers the documentation for this entire ecosystem — enabling each project to expose its API reference across Python, TypeScript, Rust, R, Julia, C#, and Go from a single Starlight configuration.


---

## Quick Start

```js
// astro.config.mjs
import { defineConfig } from 'astro/config';
import starlight from '@astrojs/starlight';
import polyglot from 'starlight-polyglot';

export default defineConfig({
  integrations: [
    starlight({
      title: 'My Project',
      plugins: [
        polyglot({
          python: {
            entryPoints: ['../../src/mylib'],
            output: 'api/python',
          },
          typescript: {
            entryPoints: ['../../src/index.ts'],
            output: 'api/typescript',
          },
        }),
      ],
      sidebar: [
        {
          label: 'Python API',
          autogenerate: { directory: 'api/python' },
        },
        {
          label: 'TypeScript API',
          autogenerate: { directory: 'api/typescript' },
        },
      ],
    }),
  ],
});
```

## When to Use Starlight Polyglot

| Use Case | Description |
|----------|-------------|
| **Polyglot Libraries** | Your library has implementations in multiple languages and you want a single docs site |
| **SDK Documentation** | Generate consistent API docs for SDKs shipped in different languages |
| **Research Software** | Share HEOR tools with collaborators who use different programming languages |
| **Internal Tooling** | Maintain a single documentation source for polyglot internal codebases |

## Key Features

<ul class="feature-list">
  <li><strong>Multi-Language Support</strong> — Generate API docs from Python (pydoc), TypeScript (TypeDoc), Rust (rustdoc), R (roxygen2), Julia (Documenter.jl), C# (DocFX), and Go (go doc).</li>
  <li><strong>Unified Sidebar</strong> — All language APIs appear in a consistent Starlight sidebar with language-specific grouping.</li>
  <li><strong>Language Tabs</strong> — Auto-generated language tabs let users switch between implementations of the same concept across languages.</li>
  <li><strong>Cross-Language Links</strong> — Link to equivalent functions across languages — e.g., a Python `evpi()` links to the TypeScript `evpi()`. Hand-written documentation links to auto-generated API references.</li>
  <li><strong>Pluggable Handlers</strong> — Each language has a handler that extracts documentation from source code. Custom handlers can be written for private/legacy codebases.</li>
  <li><strong>Customisable Output</strong> — Control the output structure, navigation labels, and page templates.</li>
</ul>
