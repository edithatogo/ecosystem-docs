## Cross-Project Workflow

The real power of this ecosystem comes from combining tools. Here's a typical **health economics workflow** that spans all projects:

### Step 1: Import and Process Real-World Data

Use **Mars** to clean and resample your real-world time-series data — for example, monthly prescription claims or hospital admission records.

```go
import "github.com/edithatogo/mars"

claims := mars.Resample(rawClaims, mars.Downsample{Factor: 1, Method: mars.Sum})
trend := mars.RollingMean(claims, 3)
```

### Step 2: Fit a Diffusion Model

Pass the processed data to **Innovate** to fit a Bass diffusion model and forecast adoption curves.

```python
from innovate import fit_model, predict_model
import pyarrow as pa

# Read Mars-processed data via Arrow
table = pa.ipc.open_file("claims.arrow").read_all()
df = table.to_pandas()

result = fit_model(df, model='bass', target_col='adoptions')
forecast = predict_model(result, horizon=12)
```

### Step 3: Build a Microsimulation Model

Use **Lifecourse** to simulate patient trajectories that incorporate the adoption forecasts as treatment uptake rates.

```python
from lifecourse import MicrosimulationModel

model = MicrosimulationModel(
    states=[...],
    adoption_curve=forecast['predicted_adoptions'],
    n_patients=50000
)
results = model.run()
```

### Step 4: Value of Information Analysis

Use **Voiage** to compute the expected value of reducing uncertainty in your model parameters, informing research priorities.

```python
from voiage import evpi, evppi

# Use lifecourse PSA outputs
evpi_value = evpi(net_benefit_samples)
```
